> 版本：v1.0

## 1. 产品概述

### 1.1 产品名称

GlobalLeads — 海外线索挖掘平台

### 1.2 产品定位

帮中国外贸/跨境电商企业，用 AI 从海外社交媒体和 B2B 渠道自动挖掘销售线索，返回潜在客户的联系方式。用户获取线索后自行触达。

### 1.3 核心原则

- **只做线索挖掘和交付**，不做邮件发送、不做 CRM
- **全部使用官方/开源免费 API**，零成本启动
- **两个功能独立**：社媒线索挖掘和 B2B 客户搜索各自独立的线索池
- **大模型可切换**：测试用本地 Ollama，上线切 DeepSeek，环境变量控制

### 1.4 目标用户

中国外贸企业、跨境电商卖家的销售团队（1-10 人），典型画像：
- 有产品但不知道海外买家在哪
- 英语能力有限，难以主动搜索海外客户
- 传统渠道（阿里国际站、展会）获客成本越来越高

## 2. 功能模块

### 2.1 社媒线索挖掘

**本质：** 在海外社交媒体上主动搜索用户发布的帖子/评论，用 AI 判断是否有购买意向。

**用户操作流程：**
1. 创建挖掘任务：输入产品关键词（如 "LED lighting"、"wholesale shoes"）+ 选择平台 + 选择语言/地区
2. 系统通过 API 搜索相关帖子/评论
3. 大模型分析内容，判断发帖人是否有购买意图
4. 返回筛选后的线索列表

**返回字段：**
| 字段 | 说明 |
|------|------|
| 作者昵称 | 发帖人用户名 |
| 作者主页链接 | 发帖人主页 URL |
| 平台来源 | Reddit / Bluesky / YouTube |
| 帖子内容 | 原始帖子/评论文本 |
| 发布时间 | 帖子发布时间 |
| AI 意向评分 | 1-100，越高越可能有购买意图 |
| 意向标签 | 如：求购、比价、找供应商、问推荐 |
| 任务 ID | 关联的挖掘任务 |

**支持的社媒平台（MVP）：**
| 平台 | 免费 API | 额度 |
|------|---------|------|
| Reddit | Reddit API | 60 请求/分钟 |
| Bluesky | AT Protocol API | 3000 请求/5 分钟 |
| YouTube | YouTube Data API v3 | 100 次搜索/天 |

**扩展平台（第二期）：**
- Mastodon
- GitHub（科技类）
- 更多平台按需接入

### 2.2 B2B 客户搜索

**本质：** 通过第三方 B2B 数据 API，按行业和地区搜索目标公司及联系人。

**用户操作流程：**
1. 创建搜索任务：选择行业 + 地区 + 公司规模（可选）
2. 系统调用第三方 API 搜索目标公司
3. 获取联系人邮箱并验证有效性
4. 返回公司和联系人列表

**返回字段：**
| 字段 | 说明 |
|------|------|
| 公司名称 | 目标公司名 |
| 公司网站 | 官网 URL |
| 公司规模 | 员工数量范围 |
| 公司地址 | 所在地区/地址 |
| 联系人姓名 | 决策者姓名 |
| 联系人职位 | 如 CEO、Purchasing Manager |
| 联系人邮箱 | 已验证的邮箱地址 |
| 行业分类 | 所属行业 |
| 数据来源 | Apollo / Google Maps / 其他 |
| 任务 ID | 关联的搜索任务 |

**支持的数据源（MVP）：**
| 数据源 | 免费额度 | 能获取 |
|--------|---------|--------|
| Apollo.io | 900 积分/月 | 公司 + 联系人 + 邮箱 |
| Google Maps Places | ~6,250 次搜索/月 | 本地商家信息 |
| Snov.io | 50 积分/月 | 邮箱查找 + 验证（补充） |
| Hunter.io | 25 积分/月 | 域名邮箱查找（补充） |

### 2.3 任务管理

用户可以管理所有挖掘/搜索任务：

| 能力 | 说明 |
|------|------|
| 创建任务 | 设置关键词/搜索条件，启动挖掘 |
| 查看任务状态 | 待处理 / 进行中 / 已完成 / 失败 |
| 查看任务结果 | 任务下的所有线索列表 |
| 停止/重试任务 | 控制任务执行 |
| 任务历史记录 | 查看历史执行的任务 |

### 2.4 线索管理

**社媒线索和 B2B 线索各自独立管理，不混合。**

| 能力 | 说明 |
|------|------|
| 线索列表 | 按来源（社媒/B2B）分别查看 |
| 筛选排序 | 按评分、时间、平台、地区筛选 |
| 线索详情 | 查看线索完整信息和来源帖子/公司详情 |
| 导出 | 导出 CSV/Excel，用户自行触达 |
| 标记状态 | 未联系 / 已联系 / 有回复 / 无效 |

### 2.5 数据看板

展示关键指标，帮助用户了解线索挖掘效果：

| 指标 | 说明 |
|------|------|
| 总线索数 | 社媒 + B2B 分别统计 |
| 本周新增线索 | 趋势图 |
| 各平台线索分布 | 饼图 |
| 意向评分分布 | 柱状图 |
| 任务执行统计 | 成功率、失败率 |
| API 用量统计 | 各数据源剩余额度 |

## 3. 不做的功能（明确排除）

| 功能 | 原因 |
|------|------|
| 邮件发送 | 用户自行处理 |
| CRM 客户管理 | 后期再做 |
| AI 开发信生成 | 不需要 |
| 域名/送达率配置 | 不涉及邮件发送 |
| 多租户/团队协作 | MVP 不需要 |
| 付费系统 | MVP 免费 |

---

## 4. 非功能需求

### 4.1 性能要求
- 任务创建后异步执行，不阻塞用户操作
- 单次任务爬取时间控制在 5 分钟以内
- 前端页面加载时间 < 2 秒

### 4.2 安全要求
- 所有 API Key 通过环境变量管理，不进代码仓库
- 代码托管在 GitHub 私有仓库
- JWT 鉴权保护 API 接口

### 4.3 兼容性
- 前端支持 Chrome、Firefox、Safari、Edge 最新版本
- 适配桌面端和移动端（响应式）
- 界面语言：中文

## 5. 用户角色

MVP 阶段只有一个角色：

| 角色 | 权限 |
|------|------|
| 管理员 | 全部功能，包括系统配置 |

后期扩展：普通用户、团队成员等。

## 6. 里程碑规划

### Phase 1 — MVP（核心功能）
- 社媒线索挖掘（Reddit + Bluesky）
- B2B 客户搜索（Apollo + Google Maps）
- 任务管理
- 线索管理（各自独立）
- 基础数据看板
- 用户认证（简单登录）

### Phase 2 — 功能扩展
- YouTube 评论挖掘
- Mastodon 挖掘
- 线索去重
- 更丰富的数据看板
- 邮箱验证增强

### Phase 3 — 商业化
- 多用户系统
- 付费套餐
- Webhook 回调
- 团队协作
