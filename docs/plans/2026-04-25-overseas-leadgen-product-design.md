# 海外线索挖掘产品方案

> 日期：2026-04-25
> 状态：方案确认阶段

---

## 一、产品定位

**一句话描述：** 帮中国外贸/跨境电商企业，用 AI 从海外社媒和 B2B 渠道自动挖掘销售线索，返回联系方式。

### 核心原则

- **只做线索挖掘和交付**，不做邮件发送、不做 CRM
- 用户拿到线索后，用自己的邮箱去触达
- 全部使用官方/开源免费 API，$0 成本启动

### 目标用户（MVP 阶段）

中国外贸企业、跨境电商卖家的销售团队（1-10 人），他们：
- 有产品但不知道海外买家在哪
- 英语能力有限，写开发信困难
- 传统渠道（阿里国际站、展会）获客成本越来越高

---

## 二、核心功能

### 功能 A：社媒线索挖掘（主动挖掘，非监控）

跟 LeadMine 一样的模式：

1. 用户创建任务，设定产品关键词（如 "LED lighting"、"wholesale shoes"）
2. 爬虫去 Twitter/Reddit 等平台**搜索**相关帖子/评论
3. 大模型分析内容，判断发帖人是否有**购买意图**
4. 返回筛选后的线索（发帖人信息、内容、意向评分）

**交付字段：**
- 发帖人昵称、主页链接
- 帖子内容
- AI 意向评分（1-100）
- 意向标签（如：求购、比价、找供应商）
- 建议触达话术（AI 生成）

### 功能 B：B2B 客户搜索

1. 用户设定条件：行业 + 地区 + 公司规模
2. 调用第三方 API 搜索目标公司
3. 获取联系人邮箱并验证
4. 返回线索列表

**交付字段：**
- 公司名称、公司网站、公司规模、地区
- 联系人姓名、职位
- 邮箱（已验证）
- 公司简介

### 统一线索池

两个功能的线索汇入统一线索列表，支持：
- 标注来源（社媒 / B2B）
- AI 意向评分排序
- 标签分类（行业、地区、意向程度）
- 导出 CSV/Excel
- AI 生成个性化开发信草稿（中文输入 → 英文输出）

## 三、技术方案

### 3.1 免费数据源（$0 成本）

#### 社媒线索挖掘

| 平台 | 免费额度 | 能做什么 |
|------|---------|---------|
| **Reddit API** | 60 请求/分钟，完全免费 | 关键词搜索帖子+评论，读用户信息 |
| **Bluesky API** | 完全免费，3000 请求/5 分钟 | 关键词搜索帖子、用户、话题 |
| **YouTube Data API v3** | 10,000 配额单位/天 | 搜索视频 + 获取评论（约100次搜索/天） |
| **Mastodon API** | 完全免费，公共 API 无需认证 | 搜索帖子、话题标签 |

#### B2B 客户搜索

| 数据源 | 免费额度 | 能做什么 |
|--------|---------|---------|
| **Apollo.io** | 900 积分/月 | 公司+联系人+邮箱 |
| **Google Maps Places API** | ~6,250 次搜索/月（$200 免费额度） | 按行业+地区搜本地商家 |
| **GitHub API** | 5,000 请求/小时 | 发现科技公司 |
| **Yelp Fusion API** | 5,000 次/天 | 搜索商家 |
| **Snov.io** | 50 积分/月 | 邮箱查找+验证（补充） |
| **Hunter.io** | 25 积分/月 | 域名邮箱查找（补充） |

### 3.2 技术栈（复用 LeadMine 架构）

| 层级 | 技术 | 复用程度 |
|------|------|---------|
| 后端 | FastAPI + SQLAlchemy (async) + Celery + Redis | 90% 复用 |
| 前端 | React + TypeScript + Ant Design Pro + Vite | 85% 复用 |
| 数据库 | PostgreSQL 16 | 95% 复用 |
| AI 分析 | DeepSeek API（按量付费，极便宜） | 模式复用，Prompt 重写 |
| 部署 | Docker Compose | 95% 复用 |

### 3.3 数据流

```
用户创建任务（关键词 / 搜索条件）
        │
        ├── 社媒任务 ──→ Reddit / Bluesky / YouTube API
        │                      │
        │               原始帖子/评论数据
        │                      │
        │               LLM 意向分析（DeepSeek）
        │                      │
        │               有意向的线索 ──┐
        │                              │
        ├── B2B 任务 ──→ Apollo / Google Maps API
        │                      │
        │               公司 + 联系人数据
        │                      │
        │               邮箱验证 ────────┐
        │                                │
        └──────────────────────→ 统一线索池
                                       │
                                AI 生成开发信草稿
                                       │
                                用户导出线索 / 复制邮箱
                                       │
                                用户用自己的邮箱发送
```

## 四、部署方案

### 4.1 架构

| 组件 | 部署位置 | 说明 |
|------|---------|------|
| **后端 API** | 阿里云服务器 | FastAPI + Celery + PostgreSQL + Redis |
| **前端** | Vercel（免费） | React SPA，全球 CDN |
| **代码仓库** | GitHub 私有仓库 | 免费无限私有 repo，代码不公开 |
| **开发环境** | 本地 Mac | 写代码、调试 |

### 4.2 开发部署流程

```
本地 Mac 开发 → git push → GitHub 私有仓库
                              │
                    ┌─────────┴─────────┐
                    ▼                   ▼
            阿里云 git pull          Vercel 自动部署
            Docker Compose 启动      （关联 GitHub 仓库）
```

### 4.3 代码安全措施

| 措施 | 说明 |
|------|------|
| GitHub 私有仓库 | 代码只有自己能看，免费 |
| .gitignore 规范 | API Key、密钥、环境变量不入库 |
| 环境变量管理 | .env 文件只在服务器上配置，不进 Git |
| Deploy Key | 服务器用只读 deploy key 拉取代码，不暴露写权限 |

### 4.4 阿里云服务器资源规划

清理后现状：40GB 磁盘（剩 7.4GB），1.9GB 内存

| 服务 | 内存占用（预估） |
|------|----------------|
| LeadMine 后端（已有） | ~300MB |
| LeadMine Celery（已有） | ~200MB |
| PostgreSQL（共用） | ~150MB |
| Redis（共用） | ~50MB |
| **新项目后端** | ~200MB |
| **新项目 Celery** | ~150MB |
| 系统+其他 | ~300MB |
| **合计** | ~1.35GB / 1.9GB |

内存会比较紧张，优化措施：
- PostgreSQL 和 Redis 两个项目共用一套
- Celery worker 设 `--concurrency=1` 限制内存
- 新项目暂不部署前端（放 Vercel）

## 五、MVP 开发计划

### 第一期：核心功能（先跑通）

1. **项目搭建** — 复用 LeadMine 架构，初始化新项目
2. **Reddit 线索挖掘** — 接入 Reddit API，实现关键词搜索 → AI 分析 → 线索返回
3. **Bluesky 线索挖掘** — 接入 Bluesky API，同上流程
4. **B2B 客户搜索** — 接入 Apollo + Google Maps API
5. **线索管理页面** — 统一线索列表、筛选、导出
6. **AI 开发信生成** — 根据线索内容自动生成英文邮件草稿

### 第二期：功能扩展

- YouTube 评论挖掘
- Mastodon 挖掘
- GitHub 科技公司搜索
- 线索自动去重
- 更丰富的数据看板

### 第三期：商业化

- 用户系统（多租户）
- 付费套餐（按线索量收费）
- Webhook 回调
- 团队协作功能
